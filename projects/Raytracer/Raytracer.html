Ray Tracer

A physically-based renderer built from scratch in C++. No graphics libraries, just math and the satisfaction of watching pixels appear one ray at a time.

Overview

Ray tracing is one of those algorithms that feels like magic when you first understand it: simulate light bouncing around a scene, and an image emerges. This project implements the core concepts from scratch.

C++ Phong Illumination Recursive Reflections Shadow Rays

Features

The renderer supports ellipsoid geometry with a fixed camera at the origin looking down the negative z-axis. Each feature was implemented incrementally, building up from basic ray-sphere intersection to a full illumination model.

Implementation

The ray tracer reads scene specifications from text files and outputs PPM images. I chose C++ for performance; images render in seconds rather than the minutes Python would have required.

The core algorithm is recursive: for each pixel, cast a ray into the scene, find the closest intersection, calculate local illumination from each light source (checking for shadows), then recurse with a reflected ray up to a depth of 3.
